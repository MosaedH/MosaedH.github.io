هذي مسودة دائمة (مو منشورة) تجمع كل شي تقدر تستخدمه في كتابة المقالات. لكل ميزة: الكود فوق والنتيجة تحته مباشرة — فلو شفت النتيجة تطلع صح، معناه الميزة شغالة.

## 1. الواجهة (Frontmatter)

كل مقال يبدأ بهذي:

```yaml
---
title: عنوان المقال
description: وصف سطر أو سطرين — يطلع في نتائج البحث وبطاقات المشاركة
pubDate: 2026-07-21
updatedDate: 2026-08-01 # اختياري، يطلع سطر "آخر تحديث"
tags:
  - policies
  - nca-ecc
draft: true # شيلها أو خلها false عشان تنشر
---
```

ثلاث قواعد مهمة:

- **اسم الملف هو الرابط.** ملف `risk-register.md` يصير `/blog/risk-register/` — فخله إنجليزي دايم، لا تسمي الملف بالعربي.
- **الوسوم لازم تكون من السجل** في `src/data/tags.ts`. لو كتبت وسم مو موجود، البناء يفشل ويقول لك الوسوم المتاحة — هذي حماية مقصودة عشان ما ينشق عندك وسمين بنفس المعنى.
- **`description` مو اختياري** — هي الي تطلع في قوقل وواتساب وتويتر.

## 2. العناوين ومعرّفاتها { #headings }

```markdown
## عنوان رئيسي

### عنوان فرعي
```

جدول المحتويات على اليسار ينبني تلقائيًا من `##` و `###` بس — العناوين الأعمق ما تدخل فيه.

كل عنوان عربي ياخذ معرّف إنجليزي تلقائي منقحر، مثلًا `## إدارة المخاطر` يصير `#idara-almkhatr`. ولو تبي تتحكم في المعرّف لعنوان ناوي تشاركه:

```markdown
## ماذا يعني النطاق عمليًا؟ { #scope-definition }
```

انتبه للمسافات: مسافة بعد `{`، والـ `#` ملاصقة للمعرّف، ومسافة قبل `}`. عنوان هذي الفقرة نفسه معرّفه `#headings` يدوي.

## 3. التأكيد والألوان

```markdown
**نص غامق** و _نص مؤكد_
```

**نص غامق** و _نص مؤكد_

هنا نقطة مهمة عن الألوان: خط IBM Plex Sans Arabic **ما فيه خط مائل** أصلًا، فلو خليناه مائل المتصفح بيزيّفه وبيطلع شكله سيء بالعربي. عشان كذا `_نص_` عندك يطلع **بلون مميز ووزن أثقل** بدل الميلان — وهذا مصدر اللون الأساسي داخل النص.

يعني عندك أربع طرق جاهزة تلوّن فيها بدون أي كود:

| الأداة        | النتيجة             |
| ------------- | ------------------- |
| `_نص_`        | لون مميز + وزن ثقيل |
| `[رابط](url)` | لون مميز + خط تحت   |
| `` `كود` ``   | خلفية رمادية        |
| `> اقتباس`    | خط جانبي ملوّن      |

## 7. الجداول

```markdown
| العمود الأول | الثاني |
| ------------ | ------ |
| قيمة         | قيمة   |
```

وتقدر تتحكم بالمحاذاة عن طريق النقطتين:

```markdown
| يسار | وسط | يمين |
| :--- | :-: | ---: |
| أ    |  ب  |    ج |
```

| يسار | وسط | يمين |
| :--- | :-: | ---: |
| أ    |  ب  |    ج |

الجداول تشتغل RTL صح تلقائيًا — العمود الأول يطلع أقصى اليمين — ولو كان الجدول عريض، يصير فيه تمرير أفقي على الجوال بدل ما يكسر الصفحة. فلا تخاف من الجداول العريضة.

## 12. الترقيم الذكي

## 13. HTML خام وألوان مخصصة

الماركداون يقبل HTML مباشرة، يعني تقدر تلوّن أي شي:

```html
<span style="color: #b91c1c">نص أحمر</span>
```

<span style="color: #b91c1c">نص أحمر</span>

**بس انتبه لهذي** — أنا ما أنصح فيها كثير لسببين: اللون الثابت ما يتغير مع الوضع الليلي فممكن يصير غير مقروء، وممكن يكسر تباين الألوان (WCAG) الي الموقع حاليًا حاصل فيه ١٠٠٪. استخدمها للاستثناءات بس، وخل التأكيد العادي `_نص_` هو الأساس.

## ما الي ما يشتغل حاليًا

هذي جربتها وما تشتغل — لو كتبتها بتطلع نص خام قدام القارئ:

| الصيغة                              | الحالة  |
| ----------------------------------- | ------- |
| `:::note` صناديق تنبيه ملوّنة       | معطّلة  |
| `$x^2$` معادلات رياضية              | معطّلة  |
| `^أس^` و `~نزول~`                   | معطّلة  |
| `[[ويكي]]` روابط أوبسيدين           | معطّلة  |
| `title="file.sh"` عنوان لكتلة الكود | ما يظهر |

الأهم فيها **صناديق التنبيه الملوّنة** (تنبيه / ملاحظة / تحذير) — هذي عادة أكثر شي يرفع جودة المقال التقني، وممكن نفعّلها لو تبي.

## نصائح خاصة بالعربية

- **المصطلح الإنجليزي** اكتبه عادي داخل الجملة — المتصفح يرتبه صح، ولو تبي توضيح حط الترجمة بين قوسين: حوكمة الأمن السيبراني (Cybersecurity Governance).
- **الأرقام** استخدم الغربية (2026) لأن الموقع كله موحّد عليها.
- **الكود داخل الجملة** آمن تمامًا — جرّب `ECC-2-9-1` وسط جملة عربية، الترتيب ما ينكسر.
- **المسودات** خلها `draft: true` لين تخلص، وتشوفها محليًا بـ `npm run dev` وما تطلع للناس.
